# Resume complet des demandes du projet E-Commerce

## 1. Informations generales de l'examen

- Cours : EWA408510 - E-Commerce and Web Application
- Type : Examen final base sur un projet
- Annee academique : 2025-2026
- Enseignant : Eric Maniraguha
- Duree : 13 jours
- Periode : du 5 juin 2026 au 17 juin 2026
- Note maximale : 40 points
- Travail demande : concevoir, developper, deployer et presenter une application web e-commerce complete.

## 2. Scenario du projet

Une entreprise locale au Rwanda veut vendre ses produits ou services en ligne. Le projet consiste a creer une plateforme e-commerce moderne permettant aux clients de :

- consulter les produits ;
- voir les details des produits ;
- ajouter des produits au panier ;
- passer une commande ;
- gerer leurs achats en ligne.

Le type de commerce est libre. Exemples possibles :

- librairie ;
- boutique de mode ;
- boutique electronique ;
- pharmacie ;
- restaurant avec commande en ligne ;
- epicerie ;
- marketplace d'artisanat ;
- autre idee e-commerce approuvee.

## 3. Exigences fonctionnelles obligatoires

### A. Interface utilisateur

- Design responsive et professionnel.
- Page d'accueil claire.
- Menu de navigation.
- Interface adaptee aux mobiles.
- Experience utilisateur simple pour parcourir le site, choisir des produits et commander.

### B. Gestion des produits

- Liste des produits.
- Page de details pour chaque produit.
- Categories de produits.
- Stockage des informations produits dans une base de donnees.

### C. Panier d'achat

- Ajouter des produits au panier.
- Supprimer des produits du panier.
- Modifier les quantites.
- Calculer automatiquement les totaux.

### D. Processus de commande

- Formulaire des informations client.
- Resume de la commande.
- Confirmation de commande.
- Enregistrement des commandes dans la base de donnees.

### E. Base de donnees

La base de donnees doit permettre de stocker au minimum :

- les produits ;
- les informations clients ;
- les commandes ;
- les details des commandes si necessaire.

Un script ou fichier de base de donnees doit etre fourni si applicable.

## 4. Exigences techniques obligatoires

### A. GitHub

- Le projet doit etre heberge sur GitHub.
- Le lien du repository GitHub doit etre soumis.
- L'historique Git doit contenir des commits significatifs.
- Les commits doivent montrer une progression reelle du developpement.

### B. Deploiement en ligne

- L'application doit etre deployee en ligne.
- Le lien de deploiement doit etre soumis.
- L'application doit etre accessible pendant l'evaluation.

### C. CI/CD

- Un pipeline CI/CD est obligatoire.
- GitHub Actions ou un outil equivalent peut etre utilise.
- Le pipeline doit automatiser au minimum une partie du processus build/test/deploy lors d'un push.
- Le rapport doit expliquer le fonctionnement du CI/CD.

### D. Docker

- L'application doit etre containerisee avec Docker.
- Un `Dockerfile` est obligatoire.
- Un `docker-compose.yml` est obligatoire si le projet contient plusieurs services, par exemple application + base de donnees.
- Une preuve du build et de l'execution Docker doit etre fournie, par exemple captures d'ecran ou commandes executees.

## 7. Bonus d'innovation possibles

Jusqu'a 5 points bonus peuvent etre obtenus avec des fonctionnalites innovantes comme :

- integration d'un moyen de paiement, par exemple Mobile Money, Stripe ou PayPal ;
- recommandations basees sur l'IA ;
- tableau de bord analytique ;
- fonctionnalites de securite avancees ;
- marketplace multi-vendeurs ;
- idee commerciale originale et creative.

## 8. Contraintes et remarques importantes

- Le projet doit etre une application web.
- Les applications desktop ne sont pas acceptees.
- Le choix du langage, du framework, de la base de donnees et de la plateforme d'hebergement est libre.
- L'etudiant doit presenter et demontrer l'application.
- Pendant la defense orale, l'etudiant doit pouvoir expliquer :
  - les choix de conception ;
  - le code ;
  - la structure de la base de donnees ;
  - le processus de deploiement ;
  - l'implementation CI/CD ;
  - le fonctionnement general de l'application.
- Le travail doit etre personnel.
- Le plagiat ou l'incapacite a expliquer le projet peut entrainer une note de zero.
- Le projet doit etre termine et soumis avant le 17 juin 2026.
